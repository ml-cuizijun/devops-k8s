# 05 可观测性与故障排查

## 排障主线

1. 看状态：`kubectl get pod -A`
2. 看事件：`kubectl get events -n <ns> --sort-by=.lastTimestamp`
3. 看详情：`kubectl describe pod <pod> -n <ns>`
4. 看日志：`kubectl logs <pod> -n <ns> --tail=200`
5. 进容器：`kubectl exec -it <pod> -n <ns> -- sh`

## 高频问题清单

- `Pending`：资源不足、调度约束冲突、PVC 未绑定。
- `CrashLoopBackOff`：启动参数/依赖/配置错误。
- `ImagePullBackOff`：镜像名、仓库权限、网络失败。
- 服务不通：Service selector 错、端口映射错、探针未就绪。

## 监控日志栈（工作常见）

- Metrics Server：基础 CPU/内存数据。
- Prometheus + Grafana：监控与告警。
- Loki/EFK：日志聚合与检索。
